# -1.  搭建说明
1.1 搭建网站（可参考www.cnblogs.com/djangocn/p/10227006.html）
	1.1.1 在网站www.python.org中下载对应版本的python（3.0以上）并安装。
	1.1.2 打开cmd 使用命令 pip install django 安装django模块。
	1.1.3 打开cmd 使用命令 pip install wfastcgi 安装wfastcgi模块。
	1.1.4 打开cmd 使用命令 wfastcgi-enable 启动wfastcgi模块，并记录下":"的内容。
	1.1.5 打开iis 添加网站，并将网站的默认文件位置设置为本程序所解压的位置。
	1.1.6 打开本程序文件夹中的web.config,将其中的scriptprocessor=""引号部分改为1.1.4复制的内容。
1.2 初始化基础数据
	1.2.1初始化价目表。打开本程序文件夹中的data/jiamubiao.csv,按照，“成品”、“产品名称”、“产品规格”、“产品价格”的格式逐行输入价目信息。
	1.2.2初始化工序表。打开本程序文件夹中的data/gongxu.csv,按照，“产品名称”、“产品各工序优先值"的格式逐行输入价目信息。（优先值用数字代表，0代表无需本工序，除此之外数字小的小组将先看到对应任务，大的将在小的小组完成后看到对应任务）
	1.2.3初始化组装表。打开本程序文件夹中的data/zuzhuangdan.csv,按照二步产品名称、一步产品1名称、一步产品2名称。
2.  使用说明
2.1   ERP系统-首页
	首页主要提供各个页面的直接导航。可以通过点击对应页面标题进入对应功能页面。
2.2   ERP系统-价格管理（首页位置为基础管理系统-价目管理）
	价格管理页面主要提供对系统默认价格进行查询、增加、修改的功能。共有产品类型，产品名称，产品规格，产品价格四个待填数据空。
	当查询某个产品时，只需要填入产品名称、产品规格（产品价格保持0不变）点击查询/修改即可；
	如要修改时，需填入已有产品名称、产品规格、产品价格，点击查询/修改即可；
	如要增加时，需要填写完整的产品类型、产品名称、产品规格、产品价格，并点击查询/修改。产品类型一般分为成品和原材料两种。
2.3  ERP系统-采购系统（测试产品）
2.3.1 ERP系统-采购系统-采购登记
	采购登记主要用于等级采购意向，采购意向通过该网页输入，系统会自动分配采购序号，并记录该条采购意向。需要输入的内容有产品类型、产品名称、产品规格、产品颜色、产品价格、产品数量。输入完毕后点击提交按钮。
2.3.2 ERP系统-采购系统-采购管理
	本页面主要提供对采购意向的删除功能，只需填写对应采购序号，即可删除对应采购意向。
2.3.3 ERP系统-采购系统-采购单
	采购单页面主要提供生成标准采购单样式，并归档记录的功能。主要待填项有采购日期、采购人员、采购序号，使用流程如下：
	先输入对应采购产品的采购序号，中间用英文逗号隔开，再输入采购人名称，采购日期，点击确认即可成功生成采购单，库存中的对应物品数量也会自动增加。
	再输入采购日期，将序号输为“无”，即可搜索历史单据。
2.4  ERP系统-销售系统
2.4.1 ERP系统-销售系统-订单登记
	本页面主要提供订单登记功能，需要输入客户名称、产品名称、产品规格、产品颜色、产品价格后点击提交。系统会自动记录订单需求。
	注意：当该产品系列，产品尺寸已在价格管理中进行预设时，无需修改产品价格中的“0”,系统将根据产品系列和产品尺寸自动搜索并匹配价格。
2.4.2 ERP系统-销售系统-订单管理
	本页面主要提供订单删除和查询功能，左半部分为删除功能，需填写客户姓名、产品系列、产品尺寸、产品颜色并点击删除按钮，右半部分为查询功能，只需填写客户姓名，即可显示该客户所有未处理订单，以及产品库存。
2.4.3 ERP系统-销售系统-销售单
	本页面主要提供标准格式销售单的生成，使用方式如下：
	1）首先输入客户名称点击搜索/确认，即可生成该客户的销售单据，并自动计算商品件数以及商品总价。
	2）打印系统生成的销售单，根据销售单提取货物并签字确认。
	3）再本系统页面填入日期以及销售人员名称，点击搜索/确认，系统将会自动将单据存档，并减少对应商品库存。
2.4.4 ERP系统-销售系统-库存管理
	本页面主要提供库存的管理功能，包括查看、增加、减少、新增、修改类型、修改价格、修改报警。具体功能如下：
	1）查看：将操作类型设置为查看，输入产品名称或产品规格或产品颜色点击确认即可查询，也可同时输入两个或三个条件。
	2）增加/减少：将操作类型设置为增加/减少，输入产品名称、产品规格、产品颜色以及产品数量点击确认即可增/减对应产品的库存。
	3）新增：将操作类型设置为新增，输入产品名称、产品规格、产品颜色、产品数量点击确认即可新增产品条目。
	4）修改类型/修改价格/修改报警：将操作类型设置为对应选项时，输入产品名称、产品规格、产品颜色、修改项，即可将对应操作类型的项更改为修改项空的内容，当报警非0时，库存低于报警值的会优先显示。
2.5  ERP系统-生产系统
2.5.1  ERP系统-生产系统-生产计划系统
	本页面主要提供生产任务计划系统，主要职能是按照gongxu.csv中预设的工序登记生产计划，并自动为其编号，之后呈现给各生产小组。本页面主要输入的有产品名称，产品规格，产品颜色，产品数量。
	1）本系统所计划产品支持一步生产产品和两步生成产品，对于两步生成产品，适用于第一步生成的产品是独立产品。并支持使用两个一步产品，合成并再加工为二步产品。
	2）二步产品和一步产品需填写的内容一致，并会按照zuzhuangdan.csv设置自动减少对应一步产品（独立的半成品）库存。
	3）附加要求时针对二步产品中的第一部产品而言，对于无附加要求的第一部产品，下达生产计划时，颜色栏保持空白，有附加要求时，附加要求应在下达任务计划时填入颜色栏。与此对应，当下达二步产品时，需要在规格栏使用规格[-[半成品一要求][-半成品二要求]]。系统即会针对性的减少具有附加要求的第一步产品。
2.5.2  ERP系统-生产系统-生产计划管理
	本页面主要提供生产任务计划删除、监控、以及修改附加要求的功能（附加要求作用见2.5.1-3）。
	1）删除功能的主要职能是按照提交的任务序号删除对应任务，如任务为二步产品任务，则会自动恢复任务所使用的第一部产品（独立的半成品）的库存。
	2）监控功能的主要作用是监控正在生产中的产品的实施进度，以及已完成工序的完成时间。
	3）修改附加要求功能的主要作用是在不影响前期工序的情况下，为后续工序添加附加要求，详见2.5.1-3
2.5.3  ERP系统-生产系统-小组任务系统
	本页面主要为各生产小组、车间提供任务提交功能。需输入的数据有，任务序号、完成数量。具体使用方法如下：
	1）批量提交功能：批量提交已完成任务单时使用该功能时，在任务序号栏用英文逗号隔开所需提交的任务序号，系统将自动按照任务序号提交所填任务，完成数量栏保持默认“0”。
	2）部分完成提交功能：当某一任务单未完成全部计划数量时使用该功能，使用时限制单次填写一个任务序号，并在完成数量栏填入真是完成数量。
